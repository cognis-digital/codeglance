<a name="why"></a>
## Why codeglance?

AI-agent onboarding niche

`codeglance` is single-purpose, scriptable, and self-hostable: point it at a target, get prioritized results in the format your workflow already speaks (table · JSON · SARIF), gate CI on it, and let agents drive it over MCP.

<a name="quick-start"></a>
## Quick start

```bash
pip install cognis-codeglance
codeglance --version
codeglance scan .                       # scan current project
codeglance scan . --format json         # machine-readable
codeglance scan . --fail-on high        # CI gate (non-zero exit)
```

<a name="ai-stack"></a>
## Use it from any AI stack

`codeglance` is interoperable with every popular way of using AI:

- **MCP server** — `codeglance mcp` (Claude Desktop, Cursor, Cognis.Studio, [uncensored-fleet](https://github.com/cognis-digital/uncensored-fleet))
- **OpenAI-compatible / JSON** — pipe `codeglance scan . --format json` into any agent or LLM
- **LangChain · CrewAI · AutoGen · LlamaIndex** — wrap the CLI/JSON as a tool in one line
- **CI / scripts** — exit codes + SARIF for non-AI pipelines
